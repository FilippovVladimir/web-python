# Лабораторная работа 1

## Тема
Клиент-серверное взаимодействие, сокеты, простейшие HTTP-сервисы на Python.

## Цель работы
Изучить основы сетевого взаимодействия с использованием библиотеки `socket` в Python. Освоить протоколы UDP и TCP, а также принципы формирования HTTP-ответов.

## Структура проекта

- `task1_udp/` - UDP клиент и сервер
- `task2_tcp_math/` - TCP клиент и сервер, вариант 1
- `task3_http_file/` - HTTP сервер, отдающий `index.html`
- `task4_chat/` - многопользовательский чат
- `task5_http_journal/` - HTTP сервер с обработкой GET/POST

## Задание 1. UDP клиент и сервер

### Постановка
Клиент отправляет серверу сообщение `Hello, server`. Сервер принимает его и выводит на своей стороне. Затем сервер отправляет клиенту ответ `Hello, client`.

### Реализация
Использован протокол UDP:
- сервер создается через `socket.AF_INET` и `socket.SOCK_DGRAM`
- сервер привязывается к адресу `127.0.0.1:9001`
- клиент отправляет сообщение через `sendto`
- сервер принимает сообщение через `recvfrom`
- ответ отправляется обратно клиенту также через `sendto`

### Результат
Сообщения успешно передаются между клиентом и сервером.

## Задание 2. TCP математическая операция

### Вариант
Номер в журнале: **33**  
Вариант: **1 - Теорема Пифагора**

### Постановка
Клиент вводит два катета `a` и `b`. Сервер принимает значения, вычисляет гипотенузу `c = sqrt(a^2 + b^2)` и отправляет результат клиенту.

### Реализация
Использован протокол TCP:
- сервер создается через `socket.SOCK_STREAM`
- сервер ожидает подключение через `listen()` и `accept()`
- клиент подключается через `connect()`
- данные передаются в виде строки с двумя числами
- сервер вычисляет ответ и отправляет результат клиенту

### Результат
Клиент получает вычисленное значение гипотенузы.

## Задание 3. HTTP сервер и файл index.html

### Постановка
Реализовать сервер, который принимает HTTP запрос и отправляет HTML-страницу, загруженную из файла `index.html`.

### Реализация
- сервер работает через TCP сокет
- сервер читает файл `index.html`
- затем формирует HTTP-ответ с заголовками и телом страницы
- браузер отображает переданный HTML

### Результат
При открытии адреса сервера в браузере загружается HTML-страница из файла.

## Задание 4. Многопользовательский чат

### Постановка
Реализовать чат, в котором несколько клиентов могут подключаться к серверу и обмениваться сообщениями.

### Реализация
- использован протокол TCP
- сервер хранит список подключенных клиентов
- для каждого клиента создается отдельный поток через `threading.Thread`
- поток ждет сообщения от конкретного клиента и пересылает их другим участникам чата

### Почему используется многопоточность
Без потоков сервер обслуживал бы клиентов по одному и ожидание сообщения одного клиента блокировало бы остальных. Благодаря многопоточности каждый клиент обрабатывается независимо.

### Результат
Несколько клиентов могут одновременно обмениваться сообщениями.

## Задание 5. GET/POST HTTP сервер

### Постановка
Нужно создать простой веб-сервер, который:
- принимает данные о дисциплине и оценке
- отображает все сохраненные записи на HTML-странице

### Реализация
- реализована обработка методов GET и POST
- данные формы разбираются на стороне сервера
- данные хранятся в памяти программы в виде списка словарей
- при GET запросе формируется HTML страница со списком оценок и формой добавления

### Структура хранения данных
Используется список словарей Python:
```python
grades = [
    {"discipline": "Python", "grade": "5"},
    {"discipline": "Math", "grade": "4"}
]
```

### Результат
Сервер принимает данные формы и отображает список дисциплин и оценок.
