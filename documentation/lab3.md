# Лабораторная работа 3

## Тема
Реализация серверной части на Django REST Framework. Документирование API.

## Предметная область
**Авиарейсы**

## Цель работы
Освоить создание backend API с использованием Django REST Framework, сериализацию моделей, настройку прав доступа и авторизацию по токенам.

## Модель данных

### Flight
Модель рейса:
- номер рейса
- авиакомпания
- город вылета
- город прилета
- время вылета
- время прилета
- тип рейса
- гейт

### Reservation
Модель бронирования:
- пользователь
- рейс
- место
- дата создания
- номер билета

### Review
Модель отзыва:
- рейс
- автор
- дата рейса
- текст отзыва
- рейтинг

## Сериализаторы
Для каждой модели реализован `ModelSerializer`:
- `FlightSerializer`
- `ReservationSerializer`
- `ReviewSerializer`

Сериализаторы преобразуют объекты Django в JSON и обратно.

## ViewSet и маршруты
Для API использованы `ModelViewSet`:
- `FlightViewSet`
- `ReservationViewSet`
- `ReviewViewSet`

Маршруты API:
- `/api/flights/`
- `/api/reservations/`
- `/api/reviews/`

## Логика доступа

### Flights
- чтение доступно всем
- создание, изменение и удаление доступны только администратору

### Reservations
- доступны только авторизованному пользователю
- пользователь видит и изменяет только свои бронирования

### Reviews
- чтение доступно всем
- создание, изменение и удаление доступны только авторизованному пользователю
- редактировать и удалять можно только свои отзывы

## Авторизация
Подключен **Djoser** и токен-авторизация.

Используемые endpoint-ы:
- `POST /api/auth/users/` - регистрация
- `POST /api/auth/token/login/` - получение токена
- `POST /api/auth/token/logout/` - выход
- `GET /api/auth/users/me/` - информация о текущем пользователе

Токен передается в заголовке:
```text
Authorization: Token <token>
```

## Тестовые данные
Для быстрого наполнения базы данных реализована management-команда:
```bash
python manage.py seed
```

Команда создает:
- 20 рейсов
- 10 бронирований
- 10 отзывов
- тестовых пользователей `user1` и `user2`
